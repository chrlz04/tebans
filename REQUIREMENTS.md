1.	Software Requirements
 
User (General - applies to Admin, Consumer, Meter Reader, and Cashier)
FR-01: Login Account
The system shall allow all registered users (Admin, Consumer, Meter Reader, and Cashier) to log in to their respective accounts by entering a valid username and password. Upon submission, the system shall validate the user's credentials by checking the entered username and password against the records stored in the Login Table. The validation process verifies whether the username exists, whether the password matches, and whether the account status is active. Upon successful authentication, the user is redirected to their role-specific dashboard. If the credentials are invalid, the system shall display an error message indicating that the login attempt was unsuccessful.
FR-02: Change Password
The system shall allow all authenticated users to change their account password through their respective Settings or Profile page. The user must enter their current password for verification, followed by a new password and a confirmation of the new password. The system shall validate the current password against the records stored in the Login Table before allowing the change. The system shall enforce password requirements such as a minimum of 8 characters, including uppercase letters, numbers, and symbols. Upon successful update, the new password is stored in the Login Table.
Admin
FR-03: View Admin Dashboard
The system shall provide the Admin with a dashboard that displays a summary of key system metrics upon login. The dashboard shall automatically retrieve and display the following information: the total number of pending disconnection requests from the Disconnection Table, a list of recently registered user accounts (including full name, user type, and registration date) from the User Table, and the total count of consumer accounts with active status from the Consumer Table. The dashboard provides the Admin with a quick overview of the system's current state without requiring navigation to separate pages.
FR-04: View Staff Accounts
The system shall allow the Admin to view a list of all staff accounts (Meter Readers and Cashiers) registered in the system. The list is retrieved from the User Table and displays each staff member's full name, user type, assigned area, and account status. The Admin may optionally search for specific staff accounts by entering a search keyword such as a name, user ID, or role. When a search is performed, the system queries the User Table and returns only the matching staff records as filtered results.
FR-05: View Consumer Accounts
The system shall allow the Admin to view a list of all consumer accounts registered in the system. The list is retrieved from the Consumer Table and displays each consumer's full name, address, meter serial number, and account status. The Admin may optionally search for specific consumer accounts by entering a search keyword such as a name, consumer ID, or area. When a search is performed, the system queries the Consumer Table and returns only the matching consumer records as filtered results.
FR-06: Manage Registered Accounts
The system shall allow the Admin to view and manage all registered accounts in the system. The Admin can review pending, active, and inactive accounts retrieved from the User Table. From this interface, the Admin may choose to activate an account by updating its Account_Status to "Active," thereby granting the user access to the system. Activation is typically performed after reviewing a newly registered account or re-enabling a previously deactivated account. The Admin may also choose to deactivate an account by updating its Account_Status to "Inactive," which revokes the user's access to the system without permanently deleting their account. Deactivation may be performed for security reasons, employee termination, or policy violations.
FR-07: Create Staff Account
The system shall allow the Admin to create new staff accounts for Meter Readers and Cashiers. The Admin must provide the required details including the staff member's first name, last name, user type (Meter Reader or Cashier), contact number, assigned area, and initial password. The system generates a unique User_ID and saves the new account to the User Table with an initial Account_Status.
FR-08: Update Staff Account
The system shall allow the Admin to update the information of existing staff accounts. The Admin can modify details such as the staff member's name, contact number, and assigned area. The system retrieves the current account details from the User Table, allows the Admin to make changes, and saves the updated information back to the User Table.
Consumer
FR-09: View Billing History
The system shall allow the Consumer to view a complete history of their electricity bills. The system retrieves all billing records associated with the Consumer's account from the Billing Table and displays them in a list format showing the bill amount, due date, reading date, and payment status for each billing period.
FR-10: View Profile
The system shall allow the Consumer to view their personal account profile information. The system retrieves the Consumer's details from the Consumer Table and displays their full name, address, meter serial number, area name, contact number, and account status. This is a read-only view that allows the Consumer to verify that their information on file is accurate.
FR-11: Payment History
The system shall allow the Consumer to view a complete history of their payment transactions. The system retrieves all payment records associated with the Consumer's account from the Payment Table and displays them in a list format showing the payment amount, payment date, payment method, and the associated bill reference. The Consumer may optionally search for specific payment transactions by entering a search keyword such as a date and bill ID. When a search is performed, the system queries the Payment Table and returns only the matching transaction records as filtered results.
Meter Reader
FR-12: View Consumer Account
The system shall allow the Meter Reader to view the detailed account information of a specific consumer. The system retrieves the consumer's details from the Consumer Table and displays their full name, address, meter serial number, area name, and account status. The Meter Reader may optionally search for a specific consumer account by entering a search keyword such as a name, consumer ID, or area. When a search is performed, the system queries the Consumer Table and returns only the matching consumer records as filtered results.
FR-13: View Consumer Bill (Meter Reader)
The system shall allow the Meter Reader to view the billing details of a specific consumer. The system retrieves the consumer's billing records from the Billing Table and displays the bill amount, due date, reading date, and payment status. This allows the Meter Reader to review billing information while performing field operations.
FR-14: Update Consumer Account
The system shall allow the Meter Reader to update the information of an existing consumer account. The Meter Reader can modify details such as the consumer's name, address, contact number, and area name. The system retrieves the current account details from the Consumer Table, allows the Meter Reader to make changes, and saves the updated information back to the Consumer Table.
FR-15: Generate Bill
The system shall allow the Meter Reader to generate an electricity bill for a consumer by inputting the meter reading data. The system retrieves the consumer's account details from the Consumer Table, including the rate type and meter serial number, to calculate the bill amount. The generated bill is then saved to the Billing Table with details including the consumer reference, bill amount, due date, reading date, and payment status. After the bill is successfully generated, the system shall automatically send an SMS notification to the consumer containing their name, bill amount, and due date. The contact information is obtained from the consumer data retrieved during bill generation.
FR-16: Create Consumer Accounts
The system shall allow the Meter Reader to create new consumer accounts in the system. The Meter Reader must provide the required details including the consumer's first name, last name, address, meter serial number, area name, and contact number. The system checks for existing accounts to prevent duplicates, generates a unique Consumer_ID, and saves the new account to the Consumer Table with an initial Account_Status.
FR-17: Display Overdue Accounts
The system shall allow the Meter Reader to view a list of all consumer accounts with overdue electricity bills. The system retrieves billing records from the Billing Table where the due date has passed and the payment status remains unpaid. The list displays the consumer's name, bill amount, due date, and the number of days overdue, enabling the Meter Reader to identify consumers who may be subject to disconnection.
FR-18: Send Disconnection Request
The system shall allow the Meter Reader to submit a disconnection request for a consumer with overdue bills. The Meter Reader selects the consumer and provides the reason for disconnection. The system retrieves the consumer's details from the Consumer Table and creates a disconnection record in the Disconnection Table with the consumer reference, request date, reason, and status. After the disconnection request is successfully submitted, the system shall automatically send an SMS notification to the consumer containing their name, disconnection date, and reason for disconnection. The contact information is obtained from the consumer data retrieved during the disconnection process.
Cashier
FR-19: View Payment Collection
The system shall allow the Cashier to view a summary of all payment transactions collected. The system retrieves payment records from the Payment Table and displays them in a list format showing the consumer name, amount paid, and payment date. The Cashier may optionally filter payment collection records by specifying a start date and end date. When a date range is applied, the system queries the Payment Table for payment records that fall within the specified period and returns only the matching results.
FR-20: View Cashier Dashboard
The system shall provide the Cashier with a dashboard that displays a summary of daily payment activity upon login. The dashboard shows key metrics including Total Collections Today, Transactions Processed, Pending Cash Remittance, and Pending Consumers to Pay. The system shall also automatically retrieve and display a list of the most recent payment transactions processed on the current day from the Payment Table, showing the payment time, consumer name, and amount paid for each entry.
FR-21: Record Payment Transaction
The system shall allow the Cashier to record a payment made by a consumer for their electricity bill. The Cashier enters the consumer's details, selects the associated bill, and records the payment amount and payment method. The system verifies the current balance from the Billing Table, saves the payment data, and generates a transaction receipt for the consumer. After the payment is recorded, the system shall automatically update the payment status of the consumer's bill in the Payment Table. The status is updated to "Paid" if the full amount is settled, or "Partial" if only a portion is paid, and the outstanding balance is adjusted accordingly.
FR-22: View Consumer Bill (Cashier)
The system shall allow the Cashier to view the billing details of a specific consumer. The system retrieves the consumer's billing records from the Billing Table and displays the bill amount, due date, reading date, and payment status. The Cashier may optionally search for specific billing records by entering a search keyword such as a consumer name, bill ID, or date. When a search is performed, the system queries the Billing Table and returns only the matching billing records as filtered results.
Non-Functional Requirements
NFR-01: Performance
The system shall load all pages and process all queries within 3 seconds under normal operating conditions. Dashboard pages shall retrieve and display summary data within 2 seconds. The system shall be capable of handling at least 50 concurrent users without noticeable performance degradation. Billing generation and payment recording operations shall complete within 5 seconds, including database write operations and SMS alert delivery.
NFR-02: Security
The system shall encrypt all user passwords using a secure hashing algorithm before storing them in the Login Table. The system shall enforce role-based access control (RBAC) to ensure that users can only access features and data relevant to their assigned role (Admin, Consumer, Meter Reader, or Cashier). The system shall log all login attempts (successful and unsuccessful) for auditing purposes. Passwords must be at least 8 characters long and include uppercase letters, numbers, and symbols.
NFR-03: Usability
The system shall provide a clean and intuitive user interface that requires minimal training for new users to navigate. Each user role shall have a dedicated dashboard that displays the most relevant information and functions for their responsibilities. The system shall use consistent navigation elements, including a sidebar menu and header, across all pages. Error messages shall be descriptive and provide clear guidance on how to resolve the issue. The system shall be accessible through modern web browsers including Google Chrome, Mozilla Firefox, and Microsoft Edge.
NFR-04: Reliability
The system shall maintain an uptime of at least 99% during business operating hours (8:00 AM to 5:00 PM). The system shall implement data backup mechanisms to prevent data loss in the event of system failure. In the event of an unexpected crash, the system shall recover to the last consistent state without data corruption. All critical transactions (billing generation and payment recording) shall be processed using database transactions to ensure data integrity and prevent partial updates.
NFR-05: Scalability
The system shall be designed to accommodate growth in the number of registered consumers, staff accounts, and transaction records without requiring major architectural changes. The database schema shall support efficient querying even as data volume increases over time. The system shall be capable of scaling from the current Tubod Electric Cooperative service area to adjacent coverage zones with minimal configuration changes.
NFR-06: Availability
The system shall be available 24/7 for consumer-facing features such as View Profile, View Billing History, and Payment History, allowing consumers to access their account information at any time. Administrative and operational features (billing, payment recording, disconnection requests) shall be available during regular business hours. 
NFR-07: Data Integrity
The system shall enforce referential integrity across all database tables to ensure that relationships between entities (User, Consumer, Billing, Payment, Disconnection, and Login) remain consistent. The system shall validate all user inputs before processing to prevent invalid or incomplete data from being stored. Duplicate consumer accounts shall be prevented by checking for existing records based on meter serial number during account creation. All financial transactions (billing and payments) shall maintain accurate records with no discrepancies between the Billing Table and Payment Table.
NFR-08: SMS Notification Delivery
The system shall deliver SMS alerts to consumers within 60 seconds of bill generation or disconnection request submission. The system shall support integration with a third-party SMS gateway service for message delivery. If an SMS fails to send, the system shall log the failure and retry the delivery up to 3 times. The SMS content shall be concise and include the consumer's name and the relevant billing or disconnection details.
NFR-09: Compatibility
The system shall be compatible with the following modern web browsers: Google Chrome (version 90 and above), Mozilla Firefox (version 88 and above), Microsoft Edge (version 90 and above), and Safari (version 14 and above). The system shall be responsive and accessible on devices with screen resolutions ranging from 1024x768 (desktop) to 375x667 (mobile). The system shall function correctly on Windows, macOS, and Linux operating systems.
NFR-10: Maintainability
The system shall be developed using a modular architecture that allows individual components to be updated or replaced without affecting the entire system. The source code shall follow consistent coding conventions and include documentation for all major functions and modules. The database schema shall be well-documented to facilitate future modifications and enhancements. The system shall maintain an error logging mechanism that records system errors with timestamps and details for troubleshooting purposes.
